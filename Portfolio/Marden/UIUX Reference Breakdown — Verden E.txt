UI/UX Reference Breakdown — Verden Energy Infrastructure Website

This concept is a premium renewable energy / sustainable infrastructure website with a cinematic, editorial feel. The design combines large-scale landscape photography, minimal typography, soft grid discipline, and data-led credibility blocks. It feels like a corporate energy company, infrastructure investor, renewable developer, or sustainability finance platform.

1. Project Manager Perspective
Product Purpose

This page presents Verden as a renewable infrastructure company focused on sustainable energy systems, solar farms, wind projects, grid infrastructure, and long-term energy development.

The website is not designed like a typical corporate brochure. It is designed as a premium investor-facing brand platform where the company must communicate:

Scale
Sustainability
Technical expertise
Infrastructure credibility
International reach
Long-term reliability
Main User Intent

The visitor is likely one of these groups:

Investors researching renewable infrastructure opportunities
Government or municipal partners
Commercial energy clients
Landowners or project partners
Corporate stakeholders
Sustainability-focused organizations

The page needs to make the visitor feel that Verden is serious, established, global, and technically capable.

Content Priority

The top priority is visual trust. The user should immediately understand that Verden operates at infrastructure scale.

The second priority is proof: the metrics, project cards, map pins, and capabilities list all support the main claim.

The third priority is conversion through soft CTAs like:

About Verden
Explore project
Contact / learn more
View core services

The CTAs are intentionally subtle, not sales-heavy.

3. Motion Graphics Developer Perspective
Motion Direction

Motion should be slow, premium, and atmospheric. Nothing should bounce or feel playful.

Recommended motion style:

Easing: cubic-bezier(0.16, 1, 0.3, 1)
Duration: 700ms–1200ms
Movement: subtle vertical reveals, soft fades, image scale
Hero Animation

On page load:

Background image slowly scales from 1.08 to 1
Header fades down from -12px
Headline reveals line by line
Subcopy fades in after headline
CTA fades up
Stats card fades in from right with blur easing

Example sequence:

0ms: hero image scale begins
200ms: nav fades in
450ms: headline line 1 reveal
650ms: headline line 2 reveal
850ms: subcopy + CTA reveal
1000ms: metrics card reveal
Scroll Animations

Project cards:

Cards fade up from 24px
Image scale reduces from 1.06 to 1
Text overlays appear after image reveal

Capabilities section:

Background image uses subtle parallax
Right card floats upward slightly on scroll
Service tags stagger in row by row

Process section:

Accordion rows reveal one after another
Numbers fade in slightly later
Small wind turbine image can use slow vertical parallax

Map section:

Map lines fade in softly
Markers pulse one by one
Active marker gets a slow glow/pulse loop
Hover Interactions

CTA buttons:

hover:
background shifts slightly brighter;
arrow moves 3px right;

Project cards:

image scale: 1 → 1.04;
overlay darkens slightly;
CTA appears stronger;

Service tags:

hover background: pale green;
border-color: pale green;

Map markers:

hover scale: 1 → 1.12;
tooltip appears above marker;
Motion Rules

Avoid:

Fast transitions
Elastic easing
Overly complex animations
Rotating cards
Neon glows
Heavy cursor effects

The brand should feel stable, engineered, and premium.

4. Graphic Designer Perspective
Visual Language

The visual direction is:

Clean energy editorial
Premium infrastructure
Quiet corporate confidence
Sustainable finance
European minimalism

It combines natural landscapes with structured corporate UI.

Color Palette

Primary colors:

Role	Color
Deep green	#003F35
Dark forest green	#062E29
Pale energy green	#DDF69A
Off-white	#F4F5F1
Soft grey	#D8DDD7
Black footer	#020403
White	#FFFFFF

The pale green is used only as an accent for CTAs, active states, and map highlights.

Typography

The typography feels like a clean modern grotesk.

Recommended fonts:

Primary: Neue Haas Grotesk, Suisse Int’l, Inter, Satoshi, or Helvetica Now
Fallback: Inter / Arial / sans-serif

Heading style:

font-weight: 400;
letter-spacing: -0.035em;
line-height: 0.98–1.08;

Body style:

font-size: 11px–14px;
line-height: 1.4–1.6;
font-weight: 400;

The page intentionally uses small body text to create a premium editorial feel.

Image Treatment

Images should be:

Cinematic
Slightly desaturated
Natural green/blue tones
Soft haze
Low contrast in some areas
Darkened with gradient overlays where text appears

Recommended image treatment:

filter: saturate(0.85) contrast(0.95);

Hero photography should feel atmospheric, with cloud/fog layers.

Buttons

Buttons are small, rectangular, and understated.

Style:

font-size: 11px;
height: 28px;
padding: 0 12px;
border-radius: 2px;
background: #DDF69A;
color: #10251F;

Arrow icon should be tiny, usually inside a small square or placed at far right.

Icon Style

Icons should be:

Thin line
Geometric
Minimal
No filled illustrative icons
Stroke width around 1px–1.5px

Logo mark should be geometric and modular.

Spacing Feel

This design relies on negative space. Do not crowd the layout.

Important rules:

Keep headings isolated with breathing room
Keep body copy small
Avoid thick borders
Use thin dividers
Keep all UI elements precise and quiet

Responsive Behavior
Desktop
Full editorial layout
12-column grid
Project cards stay asymmetric
Capabilities card overlays image
Footer columns spread horizontally
Tablet
Header spacing compresses
Project grid becomes two equal columns or stacked feature layout
Capabilities card remains overlay but reduces width
Process section becomes narrower two-column layout
Mobile

Recommended mobile behavior:

Hero height: 100svh
Navigation collapses into menu button
Headline remains large but reduced to 38–44px
Metrics card moves below hero text or becomes horizontal scroll
Project grid stacks vertically
Capabilities card moves below text inside same section
Process accordions become full-width
Map section uses simplified marker layout
Footer columns stack

Mobile hero:

.hero {
  min-height: 100svh;
  padding: 24px;
}

.hero-title {
  font-size: clamp(38px, 12vw, 56px);
}
Performance Notes

Because this page is image-heavy:

Use next/image or equivalent responsive image handling
Serve WebP/AVIF
Lazy-load below-fold images
Preload hero image
Use low-quality image placeholders
Compress map SVG
Avoid video backgrounds unless necessary
Use CSS gradients instead of heavy overlay images
Accessibility

Important accessibility requirements:

Maintain contrast on text over images
Use real buttons/links for CTAs
Accordion rows must be keyboard accessible
Map markers need accessible labels
Service tags should not rely only on color
Navigation dropdowns must support keyboard focus
Use semantic section landmarks
Rebuild Summary

This reference should be rebuilt as a premium sustainability infrastructure homepage with:

Full-screen cinematic hero
Minimal transparent navigation
Large editorial typography
Asymmetric project grid
Overlay service selector card
Clean process accordion
Dark green global impact map
Large identity-driven footer

The key to reproducing it accurately is restraint: small typography, large images, precise spacing, pale green accents, and slow cinematic motion.
