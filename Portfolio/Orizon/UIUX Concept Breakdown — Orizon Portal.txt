UI/UX Concept Breakdown — Orizon Portfolio Dashboard

This reference is a modern fintech / crypto portfolio dashboard with a strong premium SaaS feel. The design combines a high-energy purple portfolio overview header with a soft, card-based analytics area below. The layout should be recreated as a desktop-first financial dashboard, with mobile behavior planned as stacked cards.

1. Project Manager Perspective
Product Context

This screen represents a personal finance / crypto investment portfolio dashboard where a user can:

View connected wallets and exchanges.
Track asset values across platforms.
Monitor portfolio balance over time.
Review holdings.
Access AI-based investment guidance.
Add new assets, wallets, or accounts.

The page is not just informational; it is designed to make the user feel that their portfolio is active, visual, premium, and easy to understand.

Screen Purpose

The main objective is to give the user a quick snapshot of:

Current portfolio performance
Wallet-level balance changes
Total balance trend
Individual asset holdings
AI/actionable financial prompts

The top section is performance-focused.
The lower section is decision/action-focused.

Main User Intent

The user arrives here to answer:

“How is my portfolio doing right now?”

Secondary intents:

Compare wallet/exchange balances.
See which holdings are increasing.
Access AI investment guidance.
Add another portfolio item.
Move between overview and holdings.

The page should support:

Data Fetching

Use TanStack Query for all server-state.

useQuery({
  queryKey: ['portfolio', 'overview'],
  queryFn: getPortfolioOverview,
});

Recommended API response should already contain finalized values:

type PortfolioOverview = {
  totalBalance: number;
  selectedRange: '1W' | '1M' | '3M' | '6M' | 'YTD' | '1Y';
  wallets: WalletSummary[];
  holdings: Holding[];
  balanceHistory: BalancePoint[];
};

The frontend should not calculate critical financial values. It should receive:

Balance
Change amount
Change percentage
Chart-ready data
Last updated label
Permission-safe holdings
